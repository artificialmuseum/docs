Artifact Features

Metadata:

This information is displayed in the List of Artifacts and the map popups.

Some of the data below is not yet being displayed, this is just a matter of time.

  • Name - required
  • Abstract (max 100 Characters) - required
  • Description (max 500 characters) - required
  • Position (Latitude & Longitude) - required

Artifact Location

  • Name (max 30 Characters) - required
  • Abstract (max 100 Characters) - required
  • Description (max 500 characters) - required
  • Position (Latitude & Longitude) - required

Artist:

3D Artist:

  • Name - required
  • Url (Homepage or Social Network Profile) - optional

Assets:

Maximum Package Size: 10 megabytes.

This includes the gltf file and any video / audio files that should be played.

GLTF 3D File:

.gltf and .glb files are supported. Textures should be optimized and packed into the gltf file. For optimal gpu performance, make sure that the image sizes are powers of two (128, 256, 512, 1024, 2048). Textures bigger than 2048x2048 pixels can usually be resized to 1024x1024 without losing visible quality. (re)baking the textures also usually makes them smaller because the bake algorithm tries to find the smallest image it can fit the texture parts into. It's allowed to use non-quadratic images too (example; 1024x512) For optimization of images tools like optipng and sharp offer command line interfaces to quickly convert multiple images.

GLTF object position

The 3D object in the gltf should be positioned on X: 0 and Y: 0, the Z position determines how high the objects floats above the floor. The Blender units translate 1:1 to meters.

Preview image - required

width: 800px, height: 600px

Skybox image - required

360 degree image of the location the artifact is positioned at.

Should be edited to remove the photographer from the image.

Alternatively, the skybox image can also be related to the artwork and not to the location.

Video - optional

ONE object in the scene has to be named "videotarget" for this to work

Depending on the object, there can theoretically also be multiple videotargets.

Caveat: The polycount of the videotarget should be as low as possible, ~20-50 faces work nicely.

Audio - optional

Unfortunately, directional audio is broken in chromium on android, so this audio file will just play in stereo. The chrome bug that tracks progress on this is a few years old by now and unlikely to be fixed until the android audio drivers are rewritten from scratch.

Testing:

Don Mc Curdy’s GLTF Viewer is a fast and simple way to test your gltf files. Having a working gltf file in the gltf viewer means that it should also be usable in the Artificial Museum.

Additional Features:

Every feature supported by gltf2 is supported by the Artificial Museum.

For more information consult the Blender Gltf2 Documentation.

Clipping

If you want to clip parts of the scene, you can add a clipping mesh to your artifact.

Just add any mesh(es) and make sure their name includes the word "clip".

This needs to be used sensibly, visitors can move INTO clipping objects, turning the clipping off.

Example: The Wound

Videotarget

Any mesh can be a videotarget, just name it "videotarget" and the video will play on the object, using the same texture mapping as that object uses before the video gets applied.

Please note that the number of vertices drops performance almost exponentially, cubes, planes and other very simple structures work well.

Example: De.Fragmentation

Animations

All animations in the gltf file will play simultaneously.

Only a few properties are animatable:

  • Keyframe(translation, rotation, scale)
  • Shape keys
  • Armatures / skinning

Example: Fake Mirror

Clickable

Allow the visitor to start and stop the animations by clicking on a play / pause button.

Displacement Map

A Video can be displayed on a plane and be it’s own displacement map.

This is hard to control, in some cases the result can be pretty interesting.

Example: Doppeldenk

Caveats

GPS accuracy

The Geo Positioning System is quite inaccurate.

This inaccuracy stems from obstruction of the signal, for example buildings, trees and even cloudy skies can have a negative impact.

This leads to an average accuracy of 10-20 meters, and currently our artworks are spawnable in a 50 meter radius around their latitude and longitude.

Phone vendors are working on multiplexing gps hardware, which would then use more than one satellite to improve the positioning using triangulation.

Audio/Video and Gltf synchronisation

It is not possible to force the audio, video and gltf animations to play at the same time.

Videos and Audio files buffer their data and might just stop playing if the bandwidth of the visitor is too low, even if we synchronized the start of the video/audio and the animations, buffering might break that sync later..

Clipping

Visitors can walk anywhere in the scene, including the INSIDE of the clipping object, breaking the clipping in weird ways.

To avoid this problem, clipping objects should only be placed where visitors are unlikely to go to (below the ground, very far from the artifact, or high in the sky (2+ meters)) alternatively, the clipping object should be as small as possible.

Interacting with the real world

We only support one method of interaction at the moment: The visitor points their device to the ground and we find the floor level using sensors.

Once the depth api lands in browsers, which is expected sometime during 2021, we will work on adding the real world as input.

This will allow plants to grow on walls, physics that are restricted by the real world, and lots of other ways to incorporate real world data into artifacts.

Compatible Browsers

IOS

Mozilla has built a special Version of Firefox for Augmented Reality: Webxr Viewer

Android

On Android, Chromium, Brave and other Chromium based browsers support Webxr. In addition, Google Play Services for AR have to be installed.
